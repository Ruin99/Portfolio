PROJECT

응원봉 제어 시스템 : AURA

프로젝트 설명



공연장의 좌석 배치도를 불러와서 사전에 작업된 카드섹션 파일의 좌표와 공연장 좌석의 좌표와
일치하는 부분에 rgb값을 보간법을 이용하여 파일로 만들어서 응원봉에 입력하는 프로그램입니다.

개발 환경


WINDOWS 10

Visual Studio 2022

C# WPF

.NET6

개발 기간 : 3개월

적용 예시

핵심 기능


공연장의 좌석 배치도를 불러와서 좌석마다 ID를 지정해줍니다.
사전에 작업한 카드섹션 파일들을 프로그램으로 불러와서
공연장 좌석의 좌표와 카드섹션 파일의 좌표가 일치하는 곳의 색상값을
보간법을 이용하여 색상의 평균값을 계산하여 해당 좌석 파일에 시간대별로 입력하는 프로그램입니다.


영상출처 : youtube
실제 동작

Step1

공연장 좌석 배치도 불러오기


공연장의 CAD 파일을 불러와서 보여줍니다.

Step2

좌석 별 ID 지정하기


좌석마다 ID를 지정해줍니다.

Step3

사전에 작업된 카드섹션 파일을 불러옵니다.


이때 보간법을 활용하여 해당 좌표의 색상 평균값을 추출하고 그 값을 시간대 별로 각 좌석 파일에 저장합니다.

프로젝트를 참여하며 느낀점

첫 프로젝트


처음으로 배워가며 진행했던 프로젝트이기에 시행착오가 많고 C# 언어 자체도 처음 써본것이기 때문에 속도적인 부분에서 많이 느렸던 프로젝트였습니다.

그렇지만 원하는 기능을 찾아내고 분석하고 다시 작성하여 하나하나 신경써서 만든 프로그램입니다. 프로젝트를 진행하면서 느꼈던 점으로는 코딩만이 전부가 아니라는것입니다.

문제를 분석하고 필요한 정보들을 찾아내고 유지보수 측면에서도 생각해야하며 전체적인 구조를 항상 생각해 가며 프로그램을 만들어야 한다는것이 이 프로잭트를 통해 깨달을수 있었습니다.

이 프로젝트 같은 경우 공연장의 CAD파일을 불러와야 합니다. CAD파일은 정해진 주기마다 버전이 달라지는데 버전이 달라지게 되면 프로그램에 오류가 발생할 가능성이 있기에

일반적인 CAD에서 사용하는 .dwg파일 대신 항상 버전이 일정한 .dxf 포맷의 파일을 불러올수 있도록 프로그램을 작성했습니다.

이러한 문제점을 찾아냈듯 작업하는 분야에 대해서도 어느정도의 정보 수집과 학습이 필요하며 이를 바탕으로 프로그램을 작성해야 큰 문제없이 작동할것입니다.

또한 UX를 중요하게 생각해야합니다. 프로그래머는 항상 사용자의 입장에서 생각하고 UX를 통해 익숙한 레이아웃을 제공해야 사용자들이 큰 문제없이 사용할수 있을것입니다.

마지막으로 실 사용자를 자주 만나며 피드백을 주고 받으며 프로그램 하나를 만들더라도 완성도 높은 , 활용성 높은 프로그램을 만들수 있게 되었습니다.

이러한 경험을 내세워 사용자 친화적인 프로그램을 만드는 것이 저만의 규칙이자 목표로 자리 잡았습니다.
