프로그래밍이란..?

프로그램

→ 데이터 구조가 구체적이면 알고리즘이 간단해진다.
Making → Finding / UI,Source 는 분리가 되어있으나 둘다 구현해야함.

프로그래밍
→ 주어진 문제를 해결 하는 것.
→ 순서 : 요구사항 분석 -> 설계 -> 구현 - > TEST

프로그래밍 언어
→ 문제해결을 위한 도구
→ 목적을 가지고 만든 언어이기 때문에 정확한 문법을 요구한다.

자료형

DataType (자료형) : 메모리에 적재될 때의 Size 를 의미한다.
프로그램에서 숫자 , 문자와 같은 Data를 임시적으로 저장하고 사용하기위해 변수가 필요하다.
변수에 Data 를 저장하고 그 변수를 이용해 연산을 하거나 새롭게 저장한다.

● 메모리의 효율적인 사용을 위해 변수는 다양하게 존재한다.

→ 정수형
- Short : 2byte
- Int : 4byte
- Long : 4byte
- …
→ 실수형
- Float : 4byte
- Double : 8byte
- …
→ 문자형
- Char : 1byte
- …

구조체 / 공용체

구조체 (struct) 의 정의
- 서로다른 데이터형의 변수들을 하나로 묶어 사용하는 기능
- 사용자의 정의형을 만드는 방법을 제공한다.
- 다른 타입의 집합 // 배열 : 다른 타입의 집합

구조체의 장점
- 통계자료,성적등과 같이 관련있는 서로 다른 자료형을 한 덩어리(집합) 로 만들어 처리할수있다.
- 연관있는 데이터를 한 덩어리로 묶으면 프로그램 관리와 구현이 용이하다.
- 데이터의 양이 많을 때 유리하다.

공용체 (Union) 의 정의
- 공용체도 사용자가 정의한 자료형이다.
- 공용체의 맴버들은 같은 주소에 할당된다(메모리 공유)
- 공용체의 크기는 멤버중 가장 큰 멤버의 크기와 같다.

함수

함수
- 함수란 하나의 특별한 목적의 작업을 수행하기 위해 독립적으로 설계된 코드의 집합이다.
1. 반환타입 : 함수가 모든 작업을 마치고 반환하는 데이터의 타입을 명시한다.
2. 함수 이름 : 함수를 호출하기 위한 이름을 명시한다.
3. 매개변수 목록(Parameters) : 함수 호출 시에 전달되는 인수의 값을 저장할 변수들을 명시한다. (DataType_Name)
4. 함수 몸체 : 함수의 고유 기능을 수행하는 명령문의 집합이다.

함수를 사용하는 이유

- 함수를 사용하는 가장 큰 이유는 반복적인 프로그래밍을 피할 수 있기 때문이다.
- 프로그램에서 특정 작업을 여러 번 반복해야 할 때는 해당 작업을 수행하는 함수를 작성하면 된다.
- 프로그램이 필요할때마다 작성한 함수를 호출하면 해당 작업을 반복해서 수행할 수 있다.
- 프로그램을 여러 개의 함수로 나누어 작성하면 , 모듈화로 인해 전체적인 코드의 가독성이 좋아진다.
- 프로그램에 문제가 발생하거나 기능의 변경이 필요할때도 손쉽게 유지보수가 가능해진다. 보통 하나의 기능을 하나의 함수로 만드는 것이 가장 좋다.

함수 호출 시에는 여러 개의 인수를 전달할 수 있지만, 함수가 반환할 수 있는 값은 1개를 넘지 못한다. 또한 함수의 특성에 따라 인수나 반환값이 하나도 없는 함수도 존재할 수 있다.

매개변수 전달 방식

매개변수 전달 방식
→ Call By Value ( 값에 의한 전달 )
- 호출한 함수에서 전달한 인자의 값을 복사하여 호출된 함수의 입력 매개변수의 초기값을 설정하는 방식이다.
- 이러한 방식에서는 호출한 곳의 인자와 호출된 곳의 인자 값은 같지만 메모리는 독립적이다.
→ Call By Memory ( 주소에 의한 호출 )
- 포인터 변수처럼 메모리 주소 값을 갖는 데이터를 인자로 전달하는 방식이다.
- 주소에 의한 호출도 포인터 변수가 갖고 있는 값이 메모리 주소인 것이므로 이 또한 값에 의한 호출방식이다.
→ Call By Reference ( 참조에 의한 전달 )
- 호출한 함수에서 전달한 인자를 호출된 함수의 입력 매개변수가 참조하여 사용하는 방식이다.
- 호출한 곳의 인자와 호출된 곳의 인자는 같은 개체다.

배열

배열 ( Array )
- 배열이란 연관된 데이터를 하나의 변수에 그룹핑 해서 관리하기 위한 방법이다.
- 배열을 이용하면 하나의 변수에 여러 정보를 담을 수 있고, 반복문과 결합하면 많은 정보도 효율적으로 관리할 수 있다.
- 순차적으로 데이터가 저장된다.
- 배열을 사용하기 좋은 상황 : 순차열적인 데이터를 저장할 때 , 어떠한 특정 요소를 빠르게 읽어야 할 때 , 데이터의 사이즈가 급변하게 자주 변하지 않을 때 , 요소를 자주 삭제해야 하지 않을 때 사용한다.
- 장점 : 순서 즉 index가 있기 때문에 index 순서로 조회가 가능하다. 그러므로 특정 인덱인덱 조회할 때, 순차적인 데이터를 저장할 때 효과적이라 할 수 있다. 예시로는 달리기 등수 , 주식 가격등등 순서가 중요한 데이터들이 있다.
- 단점 : 처음 생성될 때 정해진 메모리값을 할당받는다. 하지만 array에 들어갈 데이터가 많아져 메모리가 더 필요해지면 라시아징 , 메모리 추가 할당이 필요하다. 또한 특정요소를 조회를 하고싶을 때 인덱스값을 모른다면 인덱스 0번, 처음부터 순차적으로 조회가 되기때문에 길이가 길다면 그만큼 조회가 느려진다. 생성되고 이후 중간에 껴 넣어야할 데이터가 많다면 비효율 적이다.

포인터

포인터 ( Pointer )
- 포인터는 주소를 저장하는 변수다.
- 메모리에는 각각의 바이트를 구분하기 위한 주소가 있다.
- 포인터를 사용할 때는 주소 값이 아니라 포인터가 어떤 변수를 가리키는지가 중요하다.
- 포인터는 다른 변수를 가리키는 변수다.
- 포인터 사용시 주의사항 : 포인터는 초기화 하고 사용하는 것이 안전하다 , 어떤 변수를 가리킬지 알 수 없으면 널 포인터로 초기화 한다 , 포인터를 안전하게 사용하려면 사용할 때 널 포인터인지 검사한다.(널 포인터가 아닌 경우에만 포인터가 가리키는 변수에 접근) , 포인터의 데이터형과 포인터가 가리키는 변수의 데이터형이 같아야 한다.
- 배열 원소를 가리키는 포인터는 배열 이름인 것처럼 사용할 수 있다.
- 배열의 원소를 가리키는 포인터는 배열의 어떤 원소든지 가리킬 수 있다.

조건문

조건문
- 조건문은 주어진 조건식의 결과에 따라 별도의 명령을 수행하도록 제어하는 명령문이다.
→ If 문 : 조건식의 결과가 참이면 주어진 명령문을 실행하며 , 거짓이면 아무것도 실행하지 않는다.
→ If / else 문 : if문과 함께 사용하는 else문은 if문과는 반대로 주어진 조건식의 결과가 거짓이면 주어진 명령문을 실행한다.
→ If / else if / else 문 : 두개의 if / else 문이 연달아 나온것이다. 조건식을 여러 개 명시할 수 있으므로 중첩된 if문을 좀 더 간결하게 표현할 수 있다.
→ Switch 문 : if / else 문과 마찬가지로 주어진 조건 값의 결과에 따라 프로그램이 다른 명령을 수행하도록 하는 조건문이다. 이러한 switch문은 if/else문보다 가독성이 더 좋으며,컴파일러가 최적화를 쉽게 할 수 있어 속도 또한 빠른 편이다. switch문의 조건 값으로는 int형으로 승격할 수 있는 값만이 사용될수있다.

반복문

반복문
- 반복문은 프로그램 내에서 똑 같은 명령을 일정 횟수만큼 반복하여 수행하도록 제어하는 명령문이다. 프로그램이 처리하는 대부분의 코드는 반복적인 형태가 많으므로 가장 많이 사용되는 제어문 중 하나다.
→ While문 : 특정 조건을 만족할 때까지 계속해서 주어진 명령문을 반복 실행한다. 우선 조건식이 참인지 판단하여 참이면 내부의 명령문을 실행한다. 이렇게 내부의 명령문을 전부 실행하고 나면 다시 조건식으로 돌아와 또 한번 참인지 판단한다. 이렇게 표현식의 검사를 통해 반복해서 실행되는 반복문을 루프라고 한다.
→ do / while 문 : while문은 루프에 진입하기 전에 먼저 조건식부터 검사한다. 하지만 do/while문은 먼저 루프를 한번 실행한 후에 조건식을 검사한다. 즉 조건식의 결과와 상관 없이 무조건 한번은 루프를 실행한다.
→ for문 : for문은 while문과 달리 자체적으로 초기식 , 조건식 , 증감식을 모두 포함하고 있는 반복문이다. 따라서 while문 보다는 좀 더 간결하게 반복문을 표현할수있다.
이때 for문을 구성하는 초기식,조건식,증감식은 각각 생략될 수 있다.
→ Until문 : until문은 조건 표현이 거짓일 때 반복 수행하는 구문이다. until문은 조건 표현 뒤에 do를 명시하고 반복 수행할 구문을 표현한다.
